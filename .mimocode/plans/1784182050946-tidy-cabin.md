# 战队展示增加队员ID + 管理员导出功能

## 需求
1. 战队展示页展示队员ID信息
2. 管理员可导出所有战队信息（含队员ID）

## 当前状态
- 展示页只显示 `member_count` 和队长昵称，不显示队员列表
- 无任何导出功能
- 展示接口 `findFeatured()` 只关联了 `captain`，未加载 `members`

---

## 实现方案

### 1. 后端：展示接口加载成员列表

**文件**: `server/src/team/team.service.ts`

修改 `findFeatured()` 方法，增加关联加载 `members` 和 `members.user`：
```typescript
relations: { captain: true, members: { user: true } }
```
同时过滤只返回 `approved` 状态的成员。

**文件**: `server/src/team/team.controller.ts`

展示接口 `GET /teams/showcase` 无需改动（数据由 service 层控制）。

---

### 2. 前端：展示页显示队员ID

**文件**: `client/src/pages/TeamsShowcase.tsx`

**卡片区域**：在成员数量下方增加一行，显示前3-4个队员的昵称/game_id，超出显示"等N人"。

**详情弹窗**：增加成员列表区域，展示每个成员的：
- 角色（队长/队员 Tag）
- 昵称
- 游戏ID (`game_ids`)

---

### 3. 后端：管理员导出接口

**文件**: `server/src/team/team.controller.ts`

新增端点：`GET /api/teams/export`
- 需要 JWT 认证 + 管理员权限检查 (`user.role === 1`)
- 查询所有战队，关联 `captain` + `members.user`
- 返回 JSON 数组（前端转 CSV 下载）

**文件**: `server/src/team/team.service.ts`

新增 `findAllWithMembers()` 方法：
- 查询所有战队
- 关联 captain 和 members（含 user）
- 按创建时间倒序

---

### 4. 前端：导出按钮

**文件**: `client/src/pages/Admin.tsx` 或 `client/src/pages/TeamsShowcase.tsx`

在管理员可见区域增加"导出战队数据"按钮，调用导出接口，前端生成 CSV 文件下载。

CSV 列：战队名称 | 标签 | 队长 | 成员数 | 成员ID列表 | 成就 | 创建时间

---

## 涉及文件

| 文件 | 改动 |
|------|------|
| `server/src/team/team.service.ts` | 修改 `findFeatured()`，新增 `findAllWithMembers()` |
| `server/src/team/team.controller.ts` | 新增 `GET /export` 端点 |
| `client/src/pages/TeamsShowcase.tsx` | 卡片和弹窗增加成员ID展示 |
| `client/src/pages/Admin.tsx` | 增加导出按钮（或在展示页加） |
| `client/src/api/index.ts` | 新增 export API 调用方法 |

## 导出按钮位置

在 `Admin.tsx` 的 Tabs 中新增"战队管理"tab，包含：
- 战队列表表格（名称、队长、成员数、成就、创建时间）
- "导出全部战队数据"按钮

---

## 验证
1. 访问 `/teams-showcase` 确认卡片和弹窗显示成员ID
2. 以管理员登录 `/admin`，切换到"战队管理"tab，点击导出
3. 确认 CSV 文件包含所有战队及成员信息
4. 非管理员访问导出接口应返回 403
